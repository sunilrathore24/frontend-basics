# Component System Design Prompt Template

## Standard Prompt Format

Use this prompt template to generate detailed system design documents for any Angular component. Simply replace `[COMPONENT_NAME]` with your desired component.

---

## PROMPT TEMPLATE:

```
I need to create an Atlassian interview-specific [COMPONENT_NAME] component for Angular.

Requirements:
1. Create a detailed system design document in markdown format
2. Include an Executive Summary with complete flow execution (high-level paragraph explaining the entire flow as if explaining to an interviewer)
3. Cover architecture, component hierarchy, data flow, and implementation details
4. Use Angular 16+ with standalone components
5. Include TypeScript interfaces and RxJS patterns
6. Cover accessibility (WCAG 2.1 AA), performance optimizations, and testing strategy
7. Include mock data structure if applicable
8. Provide keyboard navigation and user interaction patterns
9. Include SCSS styling approach
10. Add implementation phases and future enhancements

The document should be interview-ready and production-level quality.
```

## EXPECTED OUTPUT STRUCTURE:

The generated document should include:

1. **Executive Summary** - High-level flow explanation
2. **Overview** - Purpose, goals, non-goals
3. **Architecture** - Component hierarchy, tech stack
4. **Detailed Component Design** - All sub-components with TypeScript interfaces
5. **Data Flow** - RxJS streams and state management
6. **Mock Data Structure** - Sample data and filtering logic
7. **Key Features & Interactions** - Keyboard, mouse, accessibility
8. **Performance Considerations** - Optimization strategies
9. **Styling & Design** - SCSS structure and design principles
10. **Error Handling & Edge Cases**
11. **Testing Strategy** - Unit, integration, E2E tests
12. **Implementation Phases** - Week-by-week breakdown
13. **Future Enhancements** - Scalability considerations
14. **Dependencies** - Required packages
15. **API Reference** - Component usage examples
16. **Acceptance Criteria** - Functional and non-functional requirements

---

## TIPS FOR BEST RESULTS:

1. **Be Specific**: Add specific requirements if needed (e.g., "with virtual scrolling", "with lazy loading")
2. **Add Context**: Mention if it's for a specific use case (e.g., "for Jira issue management")
3. **Specify Features**: List must-have features (e.g., "with multi-select, search, and grouping")
4. **Mention Constraints**: Add any technical constraints (e.g., "must work offline", "support 10,000+ items")
